# 数据抓取标准化与系统级兼容性实施方案

该方案旨在通过统一的代码归一化规则和市场优先级策略，彻底解决指数、ETF 及个股因代码格式（前缀/后缀）导致的行情与财务数据抓取失败问题。

## 1. 数据抓取标准化规则表

无论用户输入的代码格式如何（如 `CN:ETF:159662` 或 `600519.SH`），系统将执行以下自动化处理：

| 市场区域 | 处理步骤 (代码归一化) | 行情优先接口 | 财务优先接口 | 典型转换示例 |
| :--- | :--- | :--- | :--- | :--- |
| **A股 (CN)** | 自动去掉前缀/后缀 -> 提取 6 位纯数字 | **AkShare** | **AkShare** | `CN:ETF:159662` -> `159662` |
| **港股 (HK)** | 自动去掉前缀/后缀 -> 补齐 5 位数字 | **Yahoo** | **Yahoo** | `HK:STOCK:700` -> `00700.HK` |
| **美股 (US)** | 自动去掉前缀/后缀 -> 字母大写化 | **Yahoo** | **Yahoo** | `US:ETF:SPY` -> `SPY` |

## 2. 拟议变更详细说明

### 2.1 基础抓取类重构 (`backend/data_fetcher_base.py`)

-   **核心归一化逻辑**：实现 `_normalize_raw_code` 方法，利用正则或字符串剥离逻辑去掉 `CN:STOCK:`, `HK:ETF:`, `US:INDEX:` 等前缀，以及 `.SH`, `.SZ`, `.SS`, `.HK`, `.OQ` 等后缀。
-   **抓取入口适配**：
    -   `fetch_cn_daily_data`: 归一化后执行 AkShare 逻辑。
    -   `fetch_hk_daily_data`: 归一化后执行 Yahoo 逻辑，AkShare 作为历史深度补全。
    -   `fetch_us_min_data`: 归一化后执行 Yahoo 逻辑，针对 ETF 排除 AkShare 的无效干扰。

### 2.2 财务抓取器升级 (`fetch_financials.py`)

-   **消除格式依赖**：在 `main()` 循环中，将所有的自选股 ID 统一通过归一化函数处理。
-   **接口参数补全**：根据归一化后的核心代码，结合所属市场准确定位 API 参数（如港股补全前导零，美股保持对应符号）。

### 2.3 符号配置对齐 (`backend/symbols_config.py`)

-   **典范 ID 对齐**：确保 `SYMBOLS_CONFIG` 中的所有键名已切换为标准的规范 ID（如 `SPX` 而非 `^SPX`），确保映射查询的一次性通过。

## 3. 验证与部署

1.  **连通性测试**：运行验证脚本，确认三大市场、三类标的（个股/ETF/指数）的行情与财务接口均能返回有效数据。
2.  **全量同步**：执行 `download_raw_data.py`，更新自选股池中的所有标的，确认 `MarketSnapshot` 价格补全。
3.  **结果审计**：检查 `outputs/` 目录下的 CSV 文件，确认此前缺失的标的数据已成功导出。

> [!IMPORTANT]
> 这一方案将使系统具备“代码容灾”能力。无论未来用户添加何种格式的标的，系统都能以最专业的数据源优先级进行自动对齐抓取。
