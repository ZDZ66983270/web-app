
# 数据机制与策略文档 (Data Mechanism & Strategy)

本文档详细描述了系统的数据抓取、存储、调度及代码映射策略。

---

## 1. 代码映射规则 (Symbol Mapping Rules)

为确保内部逻辑（AkShare/东财）与外部数据源（Yahoo Finance）的兼容性，请严格遵循以下规则。

### 港股市场 (HK)

| 项目 | 内部格式 (Internal) | Yahoo格式 | 规则 |
| :--- | :--- | :--- | :--- |
| **策略** | **5位数字** (补零) | **4位数字** + `.HK` | 去除前导0，但必须保留至少4位数字。 |
| **示例 1** | `00700` (腾讯) | `0700.HK` | `int(700)` -> `0700` |
| **示例 2** | `09988` (阿里) | `9988.HK` | `int(9988)` -> `9988` |
| **示例 3** | `03690` (美团) | `3690.HK` | `int(3690)` -> `3690` |
| **代码实现** | `symbol_utils.get_yahoo_symbol()` | | 详见 `backend/symbol_utils.py` |

### A股市场 (CN)

| 项目 | 内部格式 (Internal) | Yahoo格式 | 规则 |
| :--- | :--- | :--- | :--- |
| **上海** | `600xxx` | `600xxx.SS` | 后缀 `.SS` |
| **深圳** | `00xxxx`, `30xxxx` | `00xxxx.SZ` | 后缀 `.SZ` |
| **北京** | `8xxxxx`, `4xxxxx` | `xxxxxx.BJ` | 后缀 `.BJ` |

## 2. 美股市场 (US)

| 数据源 | 格式规则 | 示例 |
| :--- | :--- | :--- |
| **内部** | 股票代码 (大写) | `AAPL`, `TSLA`, `NVDA` |
| **Yahoo Finance** | 代码 + `^` (仅限指数) | `^DJI` (道指), `^NDX` (纳指) |
| **Yahoo Finance** | 代码 (股票) | `AAPL`, `TSLA` |

*注：对于个股，无需特殊转换。对于指数，Yahoo 要求 `^` 前缀。*

## 3. 数据存储策略 (Data Storage Strategy)

我们采用混合存储策略以确保数据的持久化和性能。

### A. 历史数据（静态层）
*   **来源**: 统一库 CSV 文件 (`market_data_library/history_*.csv`)。
*   **格式**: 仅日期 (`YYYY-MM-DD`)。
*   **用途**: 当添加新股票时作为"种子数据"使用。仅导入数据库一次。
*   **逻辑**:
    *   **CN 收盘**: 追加 `15:00` (例: `2023-01-01 15:00`)。
    *   **HK/US 收盘**: 追加 `16:00` (例: `2023-01-01 16:00`)。

### B. 即时数据（动态层）
*   **来源**: API (AkShare, Yahoo Finance, EastMoney)。
*   **存储**: SQLite 数据库 (`MarketDataDaily`, `MarketSnapshot`)。
*   **格式**: 完整时间戳 (`YYYY-MM-DD HH:MM:SS`)。
*   **持久化**: 所有抓取的数据通过 UPSERT (插入或更新) 存入数据库。
*   **注意**: 实时抓取 **不会** 修改 CSV 文件，只更新数据库。

## 4. 调度与频率 (Scheduling & Frequency)

### A. 首次抓取（添加时）
*   **触发**: 用户将股票添加到资产清单 (Watchlist)。
*   **动作**:
    1.  **导入历史**: 如上所述。（后台异步）
    2.  **立即抓取**: 触发 `fetch_and_save_single_stock` 获取最新的快照/实时数据。
*   **结果**: 用户能立即（或数秒内）看到数据。

### B. 周期刷新（盘中）
*   **方法**: `jobs.py` (调度器)。
*   **频率**: 可配置 (例如每 1 分钟或 15 分钟)。
*   **批量处理**:
    *   **CN (A股)**: 真批量。一次请求即可获取 5000+ 只股票数据。（极快）
    *   **HK/US**:
        *   **AkShare**: 循环处理 (逐个请求)。较慢。
        *   **Yahoo**: 支持批量 `yfinance.download(list)`。快。*推荐使用*。

### C. 15分钟快照
*   **需求**: 需要 09:30, 09:45 等特定时间点的数据。
*   **实现**:
    *   **抓取**: 通过 Yahoo/AkShare 获取 1分钟间隔数据。
    *   **处理**: 在内存或数据库视图中重采样 (Resample/Downsample) 为 15分钟频率。
    *   **原因**: 显式的 15分钟 API 可能不太稳定；1分钟 + 重采样 更稳健。

## 5. 时间戳与时区 (Timestamps & Timezones)

*   **规则**: 所有显示的时间均应反映交易发生的 **市场时间**。
*   **代码实现**: `backend/market_status.py` (类 `MarketStatus`)
*   **时区定义**:
    *   **CN**: UTC+8 (北京时间)。交易时段: 09:30 - 15:00。
    *   **HK**: UTC+8 (香港时间)。交易时段: 09:30 - 16:00。
    *   **US**: US Eastern Time (`US/Eastern`)。交易时段: 09:30 - 16:00。
*   **处理逻辑**:
    *   `is_market_open(market)`: 根据目标时区的当前时间判断是否在交易时段内。
    *   **盘中检查**: 如果当前时间处于交易时段 (`is_market_open` 为 True)，系统将严格禁止向 `MarketDataDaily` 表写入数据，防止未收盘数据污染历史记录。
*   **现象说明**: 如果您在周六 (12/14) 看到数据显示为 `12/12 16:08`，这意味着最后收到的有效数据是周四收盘的数据。这是保证数据完整性的正确行为。

## 6. 开发指南 (Development Guidelines)

*   **添加新数据源**: 必须更新 `symbol_utils.py` 以处理代码映射。
*   **测试**: 在部署前使用 `verify_*.py` 脚本验证数据流。
*   **日志**: 检查 `backend/logs_V4/` 查看抓取错误 (特别是 SSL/Timeout 错误)。
