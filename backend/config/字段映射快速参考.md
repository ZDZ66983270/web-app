## 使用示例

### Python 代码

```python
from backend.utils.field_mapper import get_field_mapper, to_db_field, map_dict_to_db

# 方式 1：使用便捷函数
db_field = to_db_field("eps_ttm")  # → "eps"

# 方式 2：使用映射器实例
mapper = get_field_mapper()
yaml_field = mapper.to_yaml_field("eps")  # → "eps_ttm"

# 方式 3：批量映射
yaml_data = {
    "eps_ttm": 1.5,
    "revenue_ttm": 1000000,
    "net_income_attributable_to_common_ttm": 500000
}
db_data = map_dict_to_db(yaml_data)
# 结果: {"eps": 1.5, "revenue_ttm": 1000000, "net_income_common_ttm": 500000}
```

## 常见问题

### Q: 为什么有些字段需要别名？
A: 历史原因导致数据库字段名与 YAML 标准名略有差异。为了向后兼容，我们使用 FieldMapper 自动处理映射，无需修改现有代码。

### Q: 如何添加新字段？
A: 
1. 在 `generic_fundamentals_core.yaml` 中添加字段定义
2. 在 `models.py` 的 `FinancialFundamentals` 类中添加对应字段
3. 如果字段名不一致，在 YAML 中设置 `status: aliased`

### Q: 银行专属字段怎么处理？
A: 银行专属字段（如 `npl_ratio`）不在通用模板中，但仍保留在数据库中。对于非银行企业，这些字段为 NULL。

### Q: 如何验证数据完整性？
A:
```python
mapper = get_field_mapper()
required_fields = mapper.list_required_fields()

# 检查缺失字段
missing = [f for f in required_fields if data.get(f) is None]
if missing:
    print(f"⚠️ Missing: {missing}")
```
